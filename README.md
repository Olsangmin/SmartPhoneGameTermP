<게임소개>
- 이 게임은 좌우의 선택버튼을 통해 제한시간이 지나기 전 계속해서 아래로 내려가는 게임이다.
- 타이밍이 맞지 않으면 실패한다.
- 가장 많은 계단을 내려가기 위한 스코어 기록형 게임이다.
- ex) 무한의 계단 -> (차이점)내려가는 형식으로 수정, 점점 배경이 올라가서 플레이어가 안보이면 실패

<참고 사항>
- 사용된 기술 : Button을 이용, xml을 이용해서 Start Button 추가
- 참고한 것들 : 유튜브) How to create an APK file in Android Studio, 스마트폰게임프로그래밍 온라인 강의, Google 검색, Chat GPT
- 수업내용에서 차용한 것 : 기본적인 배경 움직임(Dragon Flight), 캐릭터 스프라이트(Dragon Flight), 버튼을 통한 움직임(Cookie Run), 시작 이미지의 스타트 버튼(Cookie Run)
-----------------------------------------------------------------------------------------------------
<아쉬운 것들>
- 좌우 버튼을 통한 움직임
- 움직임에 따른 배경 전환
- 시간에 따른 실패여부 판별
- 자동으로 계단을 계속 생성하여 렌더
- 플레이어의 위치와 계단의 판별
- 실패 시에 사용하는 떨어지는 애니메이션

<팔기 위해 보충할 것들>
- 기본적인 구현
- 서버를 통한 점수제도
- 서버를 통한 경쟁
- 현질 아이템으로 쉬운 클리어 진행

<해결하지 못한 버그>
- 좌우 버튼 기능을 넣으면 게임이 다시 실행되는 버그
  -> 이 버그는 버그가 아니라 이해도 부족으로 느껴짐
<겪은 어려움>
- 졸업작품 중간발표, 다른 강의와의 텀프 기간 중복으로 인한 시간부족이 제일 컸다.
----------------------------------------------------------------------------------------------------
<수업에 대한 내용>
- 이번 수업에서 스마트폰을 통한 클라이언트로써 앱 개발은 어떻게 진행되는지를 알고 싶었다. 대부분의 강의는 전반적으로 이러한 내용을 다뤄서 좋았으며 기반이 자바 스크립트 언어로 구성되어 새로운 언어를 배울 기회가 많았다. 하지만 다른 언어를 배운다는 것은 결코 쉬운 일이 아니었으며 적응되어 있는 방식과는 너무도 달라 프레임워크를 이해하는데 어려움이 있었다. 다음에 다시 이런 기회가 생긴다면 더욱 열심히 임해야 겠다는 마음가짐이 생겼다.
- 이 수업에 대해서 강의자료가 많이 부족했다고 느꼈다. 특히 보통 커밋 내용으로만 공부할 방법밖에 없었기 때문에 시험 대비가 어려웠다.
- 온라인 강의의 시간이 너무 짧아서 시청기간을 놓치는 경우가 있었다. 보통 다른 수업처럼 일주일 동안이나 다음 수업 전까지 들을 수 있게 해주었으면 좋았을 것 같다.
